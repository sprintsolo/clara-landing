현재 가격 구조를 확인했습니다. 비용 구조와 함께 분석하겠습니다.

## 현재 문제: 단위 경제학이 깨져 있음

현재 원가 구조를 정리하면:

| 비용 항목 | 금액 | 비고 |
|-----------|------|------|
| 서버 | $1,000/월 (100명 기준) | = $10/user |
| LLM 토큰 | ~$200/user/월 | 가변 비용, 사용량에 비례 |
| **합계 COGS** | **~$210/user/월** | |

현재 가격과 대비하면:

| 요금제 | 현재 가격 | COGS | 마진 |
|--------|----------|------|------|
| Clara Cloud | $100/user/월 | ~$210/user | **-$110 (적자)** |
| Dedicated Server | $2,000/월 ÷ 100명 = $20/user | ~$210/user (LLM 포함 시) | **-$190 (대적자)** |

Clara Cloud는 사용자가 늘수록 손해가 커지는 구조입니다. LLM 토큰 비용($200)이 판매가($100)의 2배라서, **파는 만큼 적자**입니다.

## 핵심 전략 질문: $200 LLM 비용은 모든 사용자가 동일한가?

100인 CPA 펌을 예로 들면, 역할별 AI 사용 강도가 극적으로 다를 것입니다:

| 역할 | 인원 | AI 사용 패턴 | 추정 LLM 비용 |
|------|------|------------|-------------|
| 파트너 (10-15명) | ~12% | Engagement Intelligence 대시보드, 크로스앱 검색, Playbook 리뷰 | ~$200-300/user |
| 매니저/시니어 (30-40명) | ~35% | Playbook 생성, Client Intelligence 활용, 이메일 분석 | ~$100-150/user |
| 스태프 (50-60명) | ~53% | Playbook 소비(읽기), 기본 검색, 이메일 요약 | ~$30-50/user |

**블렌디드 평균은 $200보다 낮을 가능성이 높습니다.** 하지만 heavy user가 비용을 올리는 구조이므로, 가격 체계가 이를 반영해야 합니다.

## 제안: 3가지 가격 모델 비교

### 모델 A: 역할 기반 티어 (Role-Based Tiering)

AI 사용 강도에 따라 좌석을 3단계로 나눕니다:

| 티어 | 가격 | 대상 | AI 포함 범위 |
|------|------|------|------------|
| **Clara Essentials** | $49/user/월 | 스태프, 주니어 | Playbook 읽기, 기본 이메일 요약, 제한된 AI 검색 (월 200회) |
| **Clara Professional** | $149/user/월 | 매니저, 시니어 | Playbook 생성, Client Intelligence, Engagement Memory, AI 검색 무제한 |
| **Clara Partner** | $299/user/월 | 파트너, 디렉터 | 전체 기능 + Cross-Engagement 패턴 분석 + Engagement Intelligence 대시보드 |

**100인 CPA 펌 예시 (12 파트너 + 35 매니저 + 53 스태프):**

| 항목 | 계산 | 금액 |
|------|------|------|
| 매출 | 12×$299 + 35×$149 + 53×$49 | **$11,420/월** |
| 서버 비용 | | -$1,000 |
| LLM 비용 | 12×$250 + 35×$120 + 53×$40 | -$8,320 |
| **총 COGS** | | **-$9,320** |
| **Gross Margin** | | **$2,100 (18%)** |

문제: SaaS 표준 gross margin 70-80%에 한참 못 미칩니다. LLM 비용이 너무 지배적입니다.

### 모델 B: 플랫폼 + AI 크레딧 분리 (Platform + Consumption Hybrid)

플랫폼 접근비와 AI 사용량을 분리합니다:

| 항목 | 가격 | 포함 내용 |
|------|------|----------|
| **플랫폼 좌석** | $39/user/월 | 워크스페이스 접근, 이메일, 채팅, 프로젝트, Playbook 읽기 |
| **AI 크레딧 팩** | | |
| - Starter | $50/월 (500 크레딧) | 기본 검색, 이메일 요약 |
| - Growth | $150/월 (2,000 크레딧) | Playbook 생성, Client Intelligence |
| - Unlimited | $350/월 | 전체 기능 무제한 |

**100인 CPA 펌 예시:**

| 항목 | 계산 | 금액 |
|------|------|------|
| 플랫폼 좌석 | 100×$39 | $3,900 |
| AI 크레딧 | 12×Unlimited($350) + 35×Growth($150) + 53×Starter($50) | $12,100 |
| **총 매출** | | **$16,000/월** |

나아졌지만 여전히 70%에는 못 미칩니다. 다만 이 모델은 **사용량에 비례해 과금**하므로 실제 LLM 비용과 매출이 함께 움직이는 장점이 있습니다.

### 모델 C: Dedicated Server + BYOK (가장 현실적)

Clara가 LLM 비용을 직접 부담하지 않는 구조입니다:

## 제 권고: 모델 B + C 하이브리드

현실적으로 가장 좋은 접근은 **두 가지 트랙을 동시에 운영**하는 것입니다:

**트랙 1: Clara Cloud (모델 B) — 소규모 팀/도입 초기용**

회계펌이 처음 Clara를 시도할 때는 BYOK가 진입 장벽입니다. API 키를 발급받고 관리하는 것 자체가 IT 인프라가 약한 CPA 펌에게는 허들이니까요. 그래서 Clara Cloud는 "모든 걸 포함한 간편한 가격"으로 제공하되, AI 사용량에 따라 과금합니다.

플랫폼 좌석은 $39/user/월로 잡고, AI 크레딧을 별도로 팔되, **팀 단위 AI 예산**으로 묶습니다. 개인별이 아니라 팀 전체가 공유하는 AI 크레딧 풀을 두면, 파트너가 많이 쓰고 스태프가 적게 쓰는 자연스러운 균형이 됩니다.

| 팀 AI 플랜 | 월 가격 | 포함 크레딧 | 적합 규모 |
|-----------|--------|-----------|----------|
| Team Starter | $500/월 | 5,000 크레딧 | 5-15명 팀 |
| Team Growth | $2,000/월 | 25,000 크레딧 | 15-50명 팀 |
| Team Scale | $5,000/월 | 70,000 크레딧 | 50-100명 팀 |
| 초과 크레딧 | $0.10/크레딧 | | |

이렇게 하면 50명 팀의 경우: $39×50 + $2,000 = **$3,950/월 ($79/user)** — 도입 장벽이 낮으면서도, AI를 많이 쓸수록 자연스럽게 Team Scale로 업그레이드됩니다.

**트랙 2: Clara Dedicated (모델 C) — 100인+ 본격 도입용**

Phase 1 타겟인 100인+ CPA 펌은 어차피 IT 인프라가 있고, 데이터 보안 요구가 높습니다. BYOK가 오히려 장점이 됩니다:

| 항목 | 가격 |
|------|------|
| Dedicated Server | $3,500/월 (100명까지) |
| 추가 좌석 | $30/user/월 |
| LLM | **BYOK** (고객 직접 관리) |

고객에게 이렇게 설명합니다: "Clara Dedicated는 $35/user/월입니다. LLM 비용은 귀사의 API 키로 직접 관리하시면, 사용 패턴에 따라 평균 $80-120/user 수준입니다. **총 $115-155/user로 Engagement Intelligence를 도입**하실 수 있습니다."

이 구조의 장점은 **Clara의 gross margin이 항상 70%+ 유지**되면서, 고객 관점에서도 LLM 비용을 직접 통제할 수 있어 투명하다는 것입니다.

| 항목 | 현재 | 제안 |

| Clara Cloud | $100/user/월 + usage AI credits | **$39/user/월 + Team AI Plan ($500-5,000/월)** |

핵심 변화는 **"per-user 단일 가격"에서 "플랫폼 접근비 + AI 사용량 분리"로의 전환**입니다. 이렇게 하면 LLM 비용 변동이 Clara의 마진을 먹지 않고, 고객도 자신의 AI 사용량에 맞는 비용만 지불합니다.

이 방향에 대해 어떻게 생각하시나요? 특히 $39/user의 플랫폼 가격이 회계펌에 적절한지, 아니면 더 높여야 할지 논의하고 싶습니다.
